# NODUS — Архитектура

> Сайт — не центр. Клиент — не сервер. Ключ — не личность. Сеть — не платформа.

## Ключевой принцип

NODUS не имеет единой точки контроля, но допускает множественные точки входа.

- Официальный сайт НЕ является обязательным
- Клиент доступен через: IPFS, Tor, Git, зеркала
- Отключение любого домена не останавливает сеть

## Модель: Decentralized Core + Optional Entry Points

### Сайт (Web Entry Point)
**Назначение:** документация, загрузка клиента, onboarding

**НЕ:** хранит данные, управляет пользователями, участвует в сети

### Хостинг
- Статический сайт на IPFS
- Контент-адресация через CID
- Домен (nodus.social) — опционален, указывает на IPFS gateway
- При потере домена — сеть работает, CID публикуется публично

### Альтернативные точки входа (обязательные)
- IPFS CID
- Tor .onion адрес
- Git-репозиторий
- Community mirrors

## Клиент

**Тип:** Web App / PWA, Offline-first, Local-first

**Распространение:**
- IPFS
- Git
- Mirrors
- Direct file (index.html)

**Принципы:**
- Не требует сервера
- Не проверяет "официальность"
- Не имеет hardcoded endpoints

## Bootstrap

- Узлы только для начального соединения
- Список вшит, но заменяем и расширяем
- Узлы не авторитетны, не обязательны, не хранят данные
- Клиент запоминает peer'ов, может стартовать без bootstrap

## Сеть

- P2P (libp2p / WebRTC)
- DHT для discovery
- IPFS для контента
- E2E шифрование
- Никакой зависимости от сайта или домена

## Идентификация

- Keypair генерируется при первом запуске
- Private key хранится локально, экспортируется пользователем
- Нет аккаунтов, регистрации, подтверждений
- **Identity = ключ**

## Данные

- Хранение: локально, IPFS, между peer'ами
- Удаление: best-effort, локальное, revoke через подпись

## Безопасность

**Сайт НЕ:**
- логирует IP
- собирает cookies
- использует аналитику
- взаимодействует с сетью

**Клиент:**
- Минимизирует метаданные
- Tor/VPN friendly
- Не раскрывает реальный IP

## UI обязательные элементы

- «NODUS не имеет центрального сервера»
- «Альтернативные способы доступа»
- «Потеря ключа = потеря доступа»

## Критерий успеха

✅ Сайт можно отключить — сеть живёт  
✅ Домены можно отобрать — клиент работает  
✅ Нет entity, способного "выключить" NODUS

## Юридическая формулировка

NODUS — это протокол и программное обеспечение. Официальный сайт является лишь одним из способов распространения кода и не контролирует сеть, пользователей или данные.
